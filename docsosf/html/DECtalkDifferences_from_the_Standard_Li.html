Differences from the Standard License Management Method

The shared memory area (mapped file) has a different name.

The license check code NEVER creates the shared memory area.

If the License check does not find that the area already exists when it attempts to map it, license check fails.

The license check code never accesses the registry for any information. The license count is hardcoded at four in the DLL(s).

The license check requires that the identifier string properly decrypt. A decryption failure results in a license check failure.

The licensee uses routines in a DIGITAL-supplied source module (SMITxx.C) to initialize and close the shared memory area.
